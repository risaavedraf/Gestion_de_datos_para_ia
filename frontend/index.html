¿Por qué Lakehouse?

📋 Trazabilidad

Cada capa preserva el origen de los datos. Bronze mantiene el raw, Silver documenta las transformaciones, Gold las validaciones.

🔄 Reproducibilidad

El pipeline puede re-ejecutarse desde cualquier capa. Si cambian las reglas de validación, solo se reprocesa Gold.

🛡️ Calidad

Separación clara entre datos crudos, limpios y validados. Los rechazados se preservan para auditoría.

⚡ Performance

Parquet columnar con compresión. Lecturas 10x más rápidas que CSV. Particionamiento natural por capa.

🔒 Seguridad y Cumplimiento

🔐

PII Enmascaramiento

Datos personales (nombre, dirección, fecha de nacimiento) eliminados en capa Silver. Número de tarjeta hasheado con SHA256.

cc_num → SHA256(cc_num) → cc_num_masked
🛡️

Datos Sensibles

Columnas marcadas como sensibles: cc_num, gender, city, state, zip, lat, long, job, dob. Solo cc_num_masked persiste.

Eliminados en Silver: first, last, street, dob
📋

Ley 19.628

Ley de Protección de Datos Personales (Chile). Regula el tratamiento de datos personales y establece derechos ARCO.

Cumplimiento: Enmascaramiento, eliminación de PII, acceso restringido
📜

Ley 21.719

Ley de Protección de Datos Personales (2024). Moderniza el marco regulatorio alineándolo con GDPR.

Impacto: Consentimiento explícito, reporte de brechas, DPO
💳

PCI DSS

Payment Card Industry Data Security Standard. Requisitos para manejo seguro de datos de tarjetas de crédito.

Requisitos: Encriptar PAN, acceso mínimo necesario, auditoría
🔍

Auditoría

Trazabilidad completa: cada ejecución del pipeline se loggea con timestamp, stage, rows procesadas y checksum.

Logs: JSON estructurado en /logs/
